# Update Summary

This is a narrow amendment to the preceding bootstrap-conformance revision. It introduces no new framework control.

The revision only:

1. amends §7.10B so the three required semantic control conditions may be demonstrated by shared fixtures or mutation scenarios when every mandatory semantic acceptance contract is explicitly mapped and independently attributable;
2. harmonizes the same sharing rule in the self-test and required-demonstration sections and corresponding templates;
3. defines **mandatory semantic acceptance contract** as the contract for every mandatory field, policy, provider contract, or verification control whose satisfaction can affect PASS or FAIL, eliminating any possible narrower “load-bearing” category; and
4. removes wording that could imply this revision newly introduced all four bootstrap-era controls.

The independent-bootstrap, authorization-lineage, and separately governed conformance-suite controls are unchanged from the preceding revision.
